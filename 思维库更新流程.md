## 二、更新触发条件

| 触发条件 | 更新内容 | 更新频率 |
|---------|---------|---------|
| 做出重要决策 | 决策记录.md | 实时 |
| 发现新的思维模型 | 模型库.md | 发现时 |
| 判断失误/决策错误 | 错误教训.md | 发现时 |
| 产出优质内容 | 作品集.md | 完成时 |
| 互动中有新发现 | 记忆/YYYY-MM-DD.md | 每日 |
| 定期复盘 | 主档案 + 各子库 | 每周 |

## 三、更新流程

### 流程 1：决策后沉淀

```
1. 决策完成
   ↓
2. 填写决策记录模板
   - 决策背景
   - 思考过程
   - 最终决策
   - 预期结果
   ↓
3. 存入对应角色的决策记录.md
   ↓
4. 如果是新发现，同步到主档案
```

### 流程 2：错误后沉淀

```
1. 发现错误
   ↓
2. 填写错误教训模板
   - 错误描述
   - 根本原因
   - 修正方法
   - 防止再犯
   ↓
3. 存入错误教训.md
   ↓
4. 更新主档案（如果涉及核心原则）
```

### 流程 3：互动后沉淀

```
1. 与顾问互动结束
   ↓
2. 提取新发现（3 个问题）
   - 顾问说了什么新东西？
   - 我学到了什么？
   - 如何应用到未来？
   ↓
3. 写入记忆/YYYY-MM-DD.md
   ↓
4. 如果是重要发现，更新对应子库
```

### 流程 4：每周复盘

```
1. 回顾本周所有决策
   ↓
2. 评估决策质量
   - 哪些对了？为什么对？
   - 哪些错了？为什么错？
   ↓
3. 提炼新原则/新模型
   ↓
4. 更新主档案 + 各子库
   ↓
5. 提交 git
```

## 四、模板文件

### 决策记录模板

```markdown
## 决策记录 #XXX

**时间**: YYYY-MM-DD
**角色**: [总指挥/参谋/交易官...]
**决策类型**: [买入/战略方向/拒绝...]
**重要程度**: [高/中/低]

### 决策背景
[为什么需要做这个决策？]

### 思考过程
1. 问题是什么？
2. 有哪些选项？
3. 各选项利弊？
4. 为什么选择这个？

### 最终决策
[具体决策内容]

### 预期结果
[希望达成什么效果]

### 实际结果（复盘时填写）
[决策执行后发生了什么？对了吗？错了吗？]

### 经验教训
[从这次决策中学到什么？]
```

```markdown
### 错误描述
[发生了什么错误？]

### 根本原因
[为什么会犯这个错误？]

### 防止再犯
```

```markdown

[发现了什么？]

### 为什么重要
[这个发现有什么价值？]

### 应用场景
[在什么情况下可以用？]

### 行动建议
[需要做什么？更新档案？创建流程？]
```

## 五、沉淀规则

### 5.1 自动沉淀（无需人工干预）

| 场景 | 自动写入 |
|------|---------|
| 交易官做出投资决策 | 交易官/投资决策.md |
| 参谋发现风险 | 参谋/风险清单.md |
| 笔杆子完成内容 | 笔杆子/作品集.md |
| 进化官完成迭代 | 进化官/迭代记录.md |

### 5.2 手动沉淀（需要主动记录）

| 场景 | 何时记录 |
|------|---------|
| 发现新的思维模型 | 立即记录 |
| 判断失误 | 发现时立即记录 |
| 从互动中学到新东西 | 互动结束后记录 |
| 定期复盘发现 | 复盘时记录 |

### 5.3 定期沉淀（固定时间）

| 时间 | 沉淀内容 |
|------|---------|
| 每日结束 | 当日重要发现 → 记忆/YYYY-MM-DD.md |
| 每周复盘 | 本周决策复盘 → 各角色复盘记录 |
| 每月总结 | 提炼新原则 → 更新主档案 |

## 七、进化机制

### 7.1 原则进化

```
发现新原则
    ↓
验证原则（至少 3 个案例支持）
    ↓
写入主档案
    ↓
同步到各子库
```

### 7.2 模型进化

```
发现新思维模型
    ↓
测试模型有效性
    ↓
加入参谋/模型库.md
    ↓
在后续决策中应用
```

### 7.3 流程进化

```
发现可标准化的事
    ↓
提炼为标准流程
    ↓
写入运营官/流程库.md
    ↓
后续重复使用
```

## 九、Git 提交规范

### 提交格式

```
<type>: <subject>

<body>

<footer>
```

### Type 类型

| Type | 说明 |
|------|------|
| docs | 更新档案/文档 |
| decision | 记录决策 |
| lesson | 记录教训 |
| discovery | 记录新发现 |
| refactor | 重构思维库结构 |

### 示例

```
decision: 交易官拒绝投资某新能源公司

原因：
1. 无明显护城河
2. 价格无安全边际
3. 财务风险大

更新文件：
- 交易官/拒绝清单.md
- 交易官/投资决策.md
```

## 十、自动化建议

### 可自动化部分

1. **决策记录自动生成**
   - 每次交易官做决策后，自动生成决策记录模板
   - 填入时间、角色、决策内容

2. **每周复盘提醒**
   - 周末提醒进行复盘
   - 列出本周所有决策

3. **相似案例推荐**
   - 记录新决策时，推荐相似的历史决策
   - "上次类似情况，我们做了什么决策？"

### 需人工输入部分

1. 实际结果（复盘时填写）
2. 经验教训（主观判断）
3. 新发现的提炼

# Buy-Side News Analyst - 买方解读新闻

## 核心理念

**买方思维 vs 卖方思维**

| 维度 | 卖方思维 | 买方思维 |
|------|----------|----------|
| 目标 | 解释发生了什么 | 判断如何行动 |
| 视角 | 表面信息整理 | 深层逻辑推演 |
| 时间 | 短期热点 | 中长期价值 |
| 风险 | 回避或淡化 | 主动识别和量化 |
| 结论 | 模糊中立 | 明确可执行 |

## 分析框架

### 1. 新闻分层解构

```
L1 - 事实层：发生了什么？（Who/What/When/Where）
L2 - 背景层：为什么现在发生？（历史对比、行业周期）
L3 - 影响层：谁会受益/受损？（产业链传导、竞争格局）
L4 - 预期层：市场是否充分定价？（预期差、认知差）
L5 - 行动层：如何布局？（标的、时机、风险）
```

### 2. 历史对比分析

- **相似情境回溯**：历史上类似事件发生后的市场反应
- **周期位置判断**：当前处于行业/经济周期的哪个阶段
- **估值对比**：当前估值 vs 历史估值分位

### 3. 细节挖掘技巧

- **措辞变化**：官方表述的微妙变化（如"稳健"→"积极"）
- **数据异常**：同比/环比/预期的偏差
- **时间巧合**：多个事件的时序关联
- **信息空白**：刻意回避的内容往往更重要

### 4. 多源交叉验证

```
新闻源 A（官方）+ 新闻源 B（媒体）+ 新闻源 C（行业）
→ 提取共同点（事实）
→ 识别差异点（立场/角度）
→ 推断未明说信息（洞察）
```

## 使用方法

```bash
# 完整买方解读
python3 scripts/buy_side_analyst.py --mode full --news "新闻内容或 URL"

# 快速解读
python3 scripts/buy_side_analyst.py --mode quick --news "新闻标题"

# 深度报告（含历史对比）
python3 scripts/buy_side_analyst.py --mode deep --news "新闻内容" --history

# 输出 JSON 格式
python3 scripts/buy_side_analyst.py --mode full --json
```

## 买方分析 CheckList

### 信息质量评估
- [ ] 信息来源是否可靠？
- [ ] 是否存在利益冲突？
- [ ] 是否有一手信息源？
- [ ] 信息时效性如何？

### 逻辑推演验证
- [ ] 因果关系是否成立？
- [ ] 是否有反例？
- [ ] 二阶/三阶影响是否考虑？
- [ ] 是否存在幸存者偏差？

### 市场定价分析
- [ ] 当前价格反映多少预期？
- [ ] 共识预期是什么？
- [ ] 是否存在预期差？
- [ ] 流动性是否支持？

### 风险评估
- [ ] 最大下行风险？
- [ ] 黑天鹅可能性？
- [ ] 相关性风险？
- [ ] 退出机制？

## 典型案例

### 案例 1：地缘政治事件
**新闻**：某国宣布对某商品加征关税

**买方解读**：
1. **事实层**：关税从 X% 提升至 Y%，生效日期 Z
2. **背景层**：对比 2018 年贸易战，本次范围更窄但针对性更强
3. **影响层**：
   - 直接受损：出口企业 A、B、C
   - 间接受损：供应链企业 D、E
   - 受益：进口替代企业 F、G
4. **预期层**：市场已定价 50% 概率，若升级则有超预期下跌
5. **行动层**：做空高敞口出口商，做多进口替代龙头

### 案例 2：财报季新闻
**新闻**：某公司营收超预期但股价下跌

**买方解读**：
1. **事实层**：营收 +15%（预期 +10%），但毛利率 -2pct
2. **细节挖掘**：毛利率下降源于低价促销，非成本上升
3. **历史对比**：2019 年类似情况，后续 3 个季度利润承压
4. **预期差**：市场担心价格战持续，而非一次性因素
5. **行动层**：短期回避，等待价格战信号明确后再布局
